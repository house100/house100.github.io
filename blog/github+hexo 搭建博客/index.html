博客搭建记录

一、前期准备

1.github：用户名zhangsan1，pang：zhangsan2.github.io ，注册邮箱zhangsan3@gmail.com（zhangsan为示例名）
2.安装Git
3.安装Node.js

二、key配置

第一步：首先打开电脑文件夹，找到C:\Users\Administrator.ssh文件夹并删除

第二步：在C:\Users\Administrator\文件夹下右键打开Git Bash Here输入命令：ssh-keygen -t rsa -C github邮件地址   生成.ssh秘钥，输入后连敲三次回车

第三步：最终生成了一个新的 C:\Users\Administrator.ssh文件夹，打开这个文件夹，找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容

第四步：打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key，把复制的内容粘贴进去，title随便填，保存即可，我们的公钥就添加成功了.

第五步：检测是否设置成功：

Hi zhangsan! You’ve successfully authenticated, but GitHub does not provide shell access.
看到这个信息说明SSH已配置成功！

三、将博客部署到 github 个人主页上

1：找一个文件夹，比如在E:\文档\blog目录下安装

右键点击Git Bash here
安装命令：

2：编辑E:\文档\blog目录下的 _config.yml 文件, 在文件末尾添加如下内容：

deploy:
type: git
repository: https://github.com/zhangsan1/zhangsan2.github.io.git
branch: main

注意：其中repository: 中的内容即为 github 个人主页链接地址

第三步：在E:\文档\blog目录下, 输入命令：hexo d 将本地 blog 推送到 github仓库, 也可能需要输入 username & pwd。

推送成功后, 在浏览器中输入对应域名, 即可访问 https://zhangsan2.github.io/

五.绑定自己域名

1.向 Github Pages 仓库添加一个CNAME(一定要大写)文件

如果是用 hexo 框架搭建博客并部署到 Github Pages 上，每次

后会把你的博客所在目录下 public 文件夹里的东西都推到 Github Pages 仓库上，并且把 CNAME 文件覆盖掉，解决这个问题可以直接把 CNAME 文件添加到 source 文件夹里，这样每次推的时候就不用担心仓库里的 CNAME 文件被覆盖掉了。

4.启用 HTTPS

HTTPS 让你的网站和网站访客更安全，并且 Github 提供的这些 IP 地址自动将你的站点加入了 CDN，提高了访问速度。你还可以在 GiHub Pages 仓库的设置里勾选 ‘Enforce HTTPS’，这样所有访问你站点的请求都会走 HTTPS。
